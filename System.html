1 ภาพรวมระบบ

ระบบประกอบด้วย 2 ส่วนหลักที่ทำงานร่วมกัน

🖥️ Server

Node.js + Express + Socket.io + node:sqlite
จัดการ API, ฐานข้อมูล, และหน้าเว็บแอดมินทั้งหมด (เรียกคิว, จอแสดงผล, ตั้งค่า, รายงาน)

🖱️ Kiosk

Electron Desktop App
ตู้กดบัตรคิวสัมผัสหน้าจอสำหรับผู้ป่วย พร้อมพิมพ์บัตรคิวจริงผ่านเครื่องพิมพ์ความร้อน

💡 Kiosk เวอร์ชัน packaged (Queue2026Kiosk.exe) จะสตาร์ท Server ให้อัตโนมัติภายในตัวเอง ไม่ต้องเปิดสองโปรแกรมแยกกัน

2 ความต้องการของระบบ

รายการเวอร์ชันหมายเหตุ
Windows10 / 11ทดสอบบน Windows 10
Node.jsLTS (22+)สำคัญ ต้องติดตั้งแยกแม้ใช้ตัว Kiosk.exe ที่ build แล้ว เพราะ server ใช้ node:sqlite ที่ต้องการ Node 22+
เครื่องพิมพ์ใบเสร็จ 80mmESC/POS (USB)เช่น POS-80 — ติดตั้ง driver Windows ให้เรียบร้อยก่อน

ติดตั้ง Node.js (ถ้ายังไม่มี):

winget install OpenJS.NodeJS.LTS
⚠️ หลังติดตั้งต้องเปิด terminal ใหม่เพื่อให้ node/npm เข้า PATH

3 วิธีติดตั้ง

3.1 สำหรับใช้งานจริง แนะนำ

  1. ติดตั้ง Node.js LTS ก่อน (ดูหัวข้อ 2)
  2. ดับเบิลคลิก kiosk\dist\Queue2026Kiosk Setup x.x.x.exe
  3. ตั้งค่าตามตัวติดตั้ง (เลือกโฟลเดอร์ติดตั้งได้)
  4. เปิดแอปจาก Desktop shortcut — ระบบจะสตาร์ท Server อัตโนมัติ, เก็บฐานข้อมูลที่ %APPDATA%\kiosk\, และเปิด fullscreen

3.2 สำหรับพัฒนา/แก้ไขโค้ด

git clone <repo>
cd andamnadev
npm install          # ติดตั้ง dependencies ทุก workspace
andamnadev/
├── server/ ← Backend (Express + Socket.io)
├── kiosk/ ← Electron Kiosk app
└── package.json ← root, รวมคำสั่งทั้งสอง workspace

curl http://localhost:8888/api/rooms

5.2 ทดสอบหน้า Kiosk

  3. ควรเห็น ส่งพิมพ์แล้ว ✓ และได้ใบเสร็จทดสอบจริง (ภาษาไทยถูกต้อง)

6 Flow การทำงานของระบบ

6.1 ภาพรวมการเชื่อมต่อ

Queue2026Kiosk.exe
Electron Main Process + Renderer
spawn child process ↓
Server (node.exe ของระบบ)
Express + Socket.io · port 8888
เจ้าหน้าที่
/calling /settings (Browser)
จอแสดงผล
/display (TV)
เครื่องพิมพ์
ESC/POS raster ผ่าน USB/TCP
❓ ทำไม Server ต้องรันด้วย Node.js ของระบบ แยกจาก Electron?
Server ใช้ node:sqlite (ต้องการ Node 22+) แต่ Electron 31 ฝัง Node เวอร์ชันเก่ากว่ามาให้ — จึง spawn เป็น child process แยกโดยใช้ node.exe ของระบบจริง (kiosk/src/server-manager.js)

6.2 Flow การออกบัตรคิว (Kiosk → พิมพ์จริง)

ผู้ป่วยเลือกห้อง → เลือกช่วงเวลา
↓
POST /api/queues สร้างเลขคิวในฐานข้อมูล
↓
window.kioskAPI.printTicket(...)
↓
mode: "pdf"render HTML → บันทึก PDF
mode: "thermal"render HTML (ฟอนต์ Sarabun) → PNG
↓
spawn child process (node.exe)แปลงภาพ → ESC/POS raster (GS v 0)
↓
WinSpool API (USB) / TCP socket
🖨️ ทำไมพิมพ์เป็น "ภาพ" ไม่ใช่ "ตัวอักษร"?
เครื่องพิมพ์ใบเสร็จราคาประหยัด/โคลนจากจีน (เช่น POS-80) ส่วนใหญ่ไม่มีฟอนต์ไทยฝังในตัวเครื่อง — ถ้าส่งตัวอักษรไทยตรงๆ จะขึ้นเป็นภาษาอื่นแทน (เช่นจีน) ไม่ว่าจะตั้ง code page ใดก็ตาม จึง render เป็นภาพด้วยฟอนต์ Sarabun ในเครื่องแล้วส่งเป็นจุดภาพ (raster) ตรงๆ รับประกันว่าตัวอักษรไทยถูกต้องเสมอ
⚙️ ทำไมต้องแยก process สำหรับแปลงภาพ?
การแปลงภาพ→ESC/POS ต้องวนลูปทีละพิกเซล (หลักแสนรอบ) ถ้ารันใน Electron main process จะถูก watchdog ฆ่าทิ้งกลางทาง จึงแยกไปรันใน Node.js child process ต่างหาก (kiosk/printer/raster-worker.js)

6.3 Flow การเรียกคิว (เจ้าหน้าที่)

เจ้าหน้าที่เปิด /calling → กดเรียกคิว
↓
POST /api/queues/:id/call
↓
Server บันทึกสถานะ + emit Socket.io "queue:update"
↓
ทุก client ที่เชื่อมต่ออยู่อัปเดตทันที (จอแสดงผล ฯลฯ)+ เล่นเสียงประกาศจาก assets/audio/th/

6.4 การ Sync ค่าตั้งค่าแบบ Realtime

เมื่อแอดมินบันทึกการตั้งค่า (โปรไฟล์บัตรคิว, ห้อง, time slot) ที่หน้า /settings → server emit settings:changed → Kiosk และหน้าอื่นๆที่เปิดอยู่จะดึงค่าใหม่ทันที ไม่ต้อง refresh เอง

7 การ Build / Deploy

7.1 Build installer ใหม่

npm run build

ขั้นตอนที่เกิดขึ้นอัตโนมัติ (เรียงตามลำดับ):

  1. prebuild — bump เลขเวอร์ชัน patch (1.0.x → 1.0.x+1)
  2. bundle-server — copy server/ + npm install --omit=dev ไปไว้ที่ kiosk/server-bundle/
  3. bundle-printer-worker — copy raster-worker.js + ติดตั้ง dependencies ไปไว้ที่ kiosk/printer-worker-bundle/
  4. electron-builder — สร้างไฟล์ .exe (NSIS installer)
  5. postbuild — ลบไฟล์ installer เวอร์ชันเก่าทั้งหมด เหลือแค่ตัวล่าสุด

ผลลัพธ์: kiosk\dist\Queue2026Kiosk Setup x.x.x.exe

📦 ทำไมต้อง bundle server/printer-worker แยก? เพราะไฟล์เหล่านี้ถูกรันด้วย system node.exe (ไม่ใช่ Electron) ซึ่งอ่านไฟล์ใน app.asar ไม่ได้ — ต้องเอาออกมาไว้นอก asar ผ่าน extraResources

7.2 Build แบบไม่สร้าง installer (ทดสอบเร็ว)

cd kiosk
npm run build:dir

ได้โฟลเดอร์ kiosk\dist\win-unpacked\ รันตรงได้โดยไม่ต้องติดตั้ง

7.3 เปิดให้เข้าถึงจากเครื่องอื่นในวง LAN

Server bind 0.0.0.0 อยู่แล้ว (รับทุก IP) แต่ต้องเปิด Firewall เพิ่ม — รันใน PowerShell แบบ Administrator:

New-NetFirewallRule -DisplayName "APQueue" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 8888 -Profile Private,Domain

หา IP เครื่อง:

Get-NetIPAddress -AddressFamily IPv4 | Where-Object { $_.InterfaceAlias -notmatch "Loopback" }

หรือเปิดหน้า "ตั้งค่าระบบ Kiosk" (แตะมุมขวาบน 5 ครั้ง) — ช่อง "IP / ที่อยู่เซิร์ฟเวอร์" จะแสดง LAN IP ให้อัตโนมัติ

8 การแก้ปัญหาที่พบบ่อย

ปัญหาสาเหตุที่พบบ่อยวิธีแก้
Kiosk ขึ้น "เชื่อมต่อเซิร์ฟเวอร์ไม่ได้"ไม่ได้ติดตั้ง Node.js / มี node.exe ค้างจับ port 8888ติดตั้ง Node.js LTS หรือ kill node.exe ทั้งหมดแล้วเปิด Kiosk ใหม่
พิมพ์ไม่ออก / หน้าขาวเครื่องพิมพ์ driver ไม่รองรับ GDI printingระบบปัจจุบันใช้ raster ESC/POS อยู่แล้ว (แก้แล้ว)
พิมพ์ออกแต่เป็นภาษาอื่น (เช่นจีน)เครื่องพิมพ์ไม่มีฟอนต์ไทยในตัวระบบปัจจุบันพิมพ์เป็นภาพอยู่แล้ว — ถ้ายังเจอ ตรวจว่า build เป็นเวอร์ชันล่าสุด
ตั้งค่าระยะ/gap ติดลบแล้วไม่มีผลเวอร์ชันเก่ามีบั๊ก clamp ค่าต่ำสุดไว้ที่ 1pxอัปเดตเป็นเวอร์ชันล่าสุด
Build แล้ว npm install ค้างเครือข่ายช้า / ไฟล์ค้างจาก build ก่อนหน้าลบ kiosk/server-bundle, kiosk/printer-worker-bundle แล้ว build ใหม่
Firewall block การเข้าจากเครื่องอื่นไม่ได้เปิด inbound ruleรันคำสั่งในหัวข้อ 7.3 แบบ Administrator
