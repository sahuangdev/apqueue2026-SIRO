# วิธีติดตั้งระบบคิว APQueue ใช้งานจริง (หลายเครื่อง)

คู่มือนี้สำหรับติดตั้งระบบคิวที่หน้างานจริง โดยกระจายเป็น **3 บทบาทเครื่อง** บนวง LAN เดียวกัน

> ถ้าต้องการเฉพาะการตั้งค่า **เครื่องจอแสดงผล** ดูคู่มือแยก: [วิธีตั้งค่าเครื่องจอแสดงผล.md](วิธีตั้งค่าเครื่องจอแสดงผล.md)

---

## 0. ภาพรวม: อะไรอยู่เครื่องไหน

```
        เครื่อง A : แม่ข่าย (server) + Kiosk ในตัว
        ├─ รันโปรแกรมหลัก (Node.js) พอร์ต 8888  ← หัวใจของระบบ
        ├─ เก็บฐานข้อมูลคิวทั้งหมด (server/data/queue.db)
        ├─ เสิร์ฟทุกหน้าเว็บให้เครื่องอื่น
        └─ ต่อเครื่องพิมพ์ความร้อน (USB) พิมพ์บัตรคิว
                              │
                       วง LAN เดียวกัน (เช่น 192.168.1.x)
        ┌─────────────────────┴───────────────────────┐
   เครื่อง B : จอแสดงผล + เสียง            เครื่อง C, D... : ประจำห้อง (เรียกคิว)
   ทีวี/จอ เปิด Chrome เต็มจอ              คอมเจ้าหน้าที่แต่ละห้อง
   .../display/  + เสียงเรียกอัตโนมัติ      .../calling  ล็อกอิน เลือกห้อง กดเรียก
```

| สิ่งที่ทำ | ทำที่เครื่องไหน | ทำซ้ำทุกเครื่องไหม |
|---|---|---|
| รันโปรแกรมหลัก + ฐานข้อมูล | เครื่อง A (server) | ❌ เครื่องเดียว |
| พิมพ์บัตรคิว (Kiosk) | เครื่อง A (ในตัว) | ❌ |
| แสดงจอคิว + เสียงเรียก | เครื่อง B | ✅ ทุกจอ |
| เรียกคิวประจำห้อง | เครื่อง C, D... | ✅ ทุกห้อง |
| ตั้งค่าระบบ (ห้อง/เสียง/รหัสผ่าน) | เปิดจากเครื่องไหนก็ได้ผ่านเว็บ | ❌ ตั้งครั้งเดียวที่ฐานข้อมูล |

**สิ่งที่ต้องเตรียม:** เครื่อง A 1 เครื่อง (สเปกทั่วไปพอ), เครื่องพิมพ์ความร้อน (ESC/POS) ต่อ USB, เครื่อง/จอแสดงผล, คอมประจำห้อง, และ **เราเตอร์/สวิตช์วงเดียวกัน** ทุกเครื่องต้องอยู่ LAN เดียวกัน

## A. ติดตั้งเครื่องแม่ข่าย (server) + Kiosk — เครื่อง A

### A1. ติดตั้ง Node.js 24
ดาวน์โหลดและติดตั้งจาก https://nodejs.org (เลือกเวอร์ชัน **24** ขึ้นไป — ระบบใช้ฐานข้อมูลในตัวของ Node ที่ต้องใช้เวอร์ชันนี้)

ตรวจว่าติดตั้งสำเร็จ: เปิด **Command Prompt** พิมพ์
```
node --version
```
ต้องขึ้น `v24.x.x` (หรือสูงกว่า)

### A2. คัดลอกโฟลเดอร์โปรเจกต์มาวาง
คัดลอกโฟลเดอร์ทั้งก้อน (`queue_2026`) มาวางที่เครื่อง A เช่น `C:\queue_2026`

> **เครื่องนี้ไม่มีอินเทอร์เน็ต?** ให้คัดลอกโฟลเดอร์ `node_modules` (ที่รากโปรเจกต์) มาด้วย — ระบบนี้ไม่มีส่วนที่ต้องคอมไพล์ จึงคัดลอกใช้ข้ามเครื่องได้เลย
> **เครื่องนี้มีอินเทอร์เน็ต?** ข้ามได้ ตอนเปิดครั้งแรก `start-server.cmd` จะรัน `npm install` ให้เอง

### A3. ตั้งค่า IP ของเครื่อง server (สำคัญ)
เครื่องอื่น (จอ/ห้อง) ต้องชี้มาที่ IP ของเครื่อง A ดังนั้นควรให้ IP **คงที่ ไม่เปลี่ยน**

- หา IP ปัจจุบัน: เปิด Command Prompt พิมพ์ `ipconfig` ดูค่า **IPv4 Address** ของการ์ดเน็ตที่ใช้งาน เช่น `192.168.1.50`
- แนะนำตั้งเป็น **Static IP** (ตั้งที่ Windows: Network Settings → IP assignment → Manual) หรือจองในเราเตอร์ (DHCP Reservation) เพื่อกัน IP เปลี่ยนแล้วเครื่องอื่นต่อไม่ติด

> จด IP นี้ไว้ จะใช้ตั้งค่าเครื่องจอและเครื่องประจำห้อง สมมุติคู่มือนี้ใช้ `192.168.1.50`

### A4. เปิดพอร์ต 8888 ใน Windows Firewall
ให้เครื่องอื่นเข้าถึง server ได้ — เปิด **Command Prompt (Run as administrator)** แล้ววางคำสั่ง:
```
netsh advfirewall firewall add rule name="Queue2026 Server 8888" dir=in action=allow protocol=TCP localport=8888
```
(หรือทำผ่านหน้าจอ: Windows Defender Firewall → Advanced → Inbound Rules → New Rule → Port → TCP 8888 → Allow)

### A5. เปิด server
ดับเบิลคลิก **`start-server.cmd`** (อยู่ที่รากโปรเจกต์)
- ครั้งแรกจะสร้างฐานข้อมูลและข้อมูลตั้งต้นให้อัตโนมัติ (ห้อง L6/L8, ช่วงเวลา, ผู้ดูแล `admin`)
- หน้าต่างจะแสดง **IP ของเครื่องนี้** และลิงก์สำหรับเครื่องจอ/เครื่องห้อง
- ถ้า server หลุด จะเปิดใหม่ให้เองอัตโนมัติ
- ทดสอบ: เปิดเบราว์เซอร์ที่เครื่อง A ไปที่ `http://localhost:8888` ต้องเห็นหน้าแดชบอร์ด

### A6. ตั้งให้ server เปิดอัตโนมัติตอนเปิดเครื่อง
1. กด `Win + R` พิมพ์ `shell:startup` แล้ว Enter → เปิดโฟลเดอร์ Startup
2. คลิกขวาที่ `start-server.cmd` → **Create shortcut** (สร้างทางลัด)
3. ลากไฟล์ทางลัดไปวางในโฟลเดอร์ Startup

ครั้งต่อไปเปิด/รีสตาร์ตเครื่อง A → server จะเปิดเองอัตโนมัติ

### A7. ติดตั้ง Kiosk (พิมพ์บัตรคิว) บนเครื่อง A
เครื่อง A ต่อเครื่องพิมพ์ความร้อนผ่าน **USB** และพิมพ์บัตรคิวให้ลูกค้า

**ขั้นที่ 1 — ติดตั้งเครื่องพิมพ์ใน Windows**
- ต่อสาย USB + ติดตั้ง driver ของเครื่องพิมพ์ความร้อน
- ตั้งค่า → Bluetooth & devices → Printers → จด**ชื่อเครื่องพิมพ์** ที่ Windows ตั้งให้ (เช่น `POS-58` หรือ `EPSON TM-T82`)

**ขั้นที่ 2 — ตั้งค่า Kiosk ให้ชี้เครื่องพิมพ์**
เปิดไฟล์ `kiosk\config.json` ด้วย Notepad แก้ให้เป็น (ใส่ชื่อเครื่องพิมพ์จริงแทน `<ชื่อเครื่องพิมพ์>`):
```json
{
  "serverUrl": "http://localhost:8888",
  "kioskId": "kiosk-1",
  "fullscreen": true,
  "width": 768,
  "height": 1024,
  "printer": {
    "mode": "thermal",
    "type": "epson",
    "interface": "printer:<ชื่อเครื่องพิมพ์>",
    "characterSet": "THAI",
    "width": 48
  }
}
```
- `serverUrl` = `localhost` เพราะ Kiosk อยู่เครื่องเดียวกับ server
- `interface` รูปแบบ `printer:ชื่อเครื่องพิมพ์ใน Windows` (เครื่องพิมพ์ USB/แชร์)
- `type` ตามยี่ห้อ: `epson` / `star` / `brother` / `tanca` / `daruma`

**ขั้นที่ 3 — เปิด Kiosk**
ดับเบิลคลิก `kiosk\start-kiosk.cmd`
- เปิดหน้าตั้งค่าเครื่องพิมพ์/ทดสอบพิมพ์: กด `Ctrl + Shift + P`
- ออกจากแอป: กด `Ctrl + Shift + Q`

**ขั้นที่ 4 — ตั้งเปิด Kiosk อัตโนมัติ** (ทำแบบเดียวกับ A6): สร้าง shortcut ของ `kiosk\start-kiosk.cmd` ไปวางใน `shell:startup`

> **ยังไม่มีเครื่องพิมพ์ ทดสอบก่อน?** ตั้ง `"mode": "pdf"` แทน — บัตรจะถูกบันทึกเป็นไฟล์ PDF ในโฟลเดอร์ `kiosk\tickets-pdf`
>
> **ทางเลือก:** มีตัวติดตั้งสำเร็จรูป [Queue2026Kiosk Setup 1.0.0.exe](kiosk/dist/Queue2026Kiosk%20Setup%201.0.0.exe) แต่การแก้ค่าเครื่องพิมพ์หลังติดตั้งทำได้ไม่สะดวกเท่ารันจาก source ข้างต้น จึง**แนะนำวิธีรันจาก source**

## B. ติดตั้งเครื่องจอแสดงผล + เสียงเรียก — เครื่อง B

ทำตามคู่มือ [วิธีตั้งค่าเครื่องจอแสดงผล.md](วิธีตั้งค่าเครื่องจอแสดงผล.md) โดยสรุป:

1. ติดตั้ง **Google Chrome** (ถ้าไม่มีจะใช้ Edge แทนอัตโนมัติ)
2. คัดลอกไฟล์ **`start-display.cmd`** มาที่เครื่อง B (วางที่ Desktop ก็ได้)
3. เปิด `start-display.cmd` ด้วย Notepad แก้ URL ให้ชี้ไปที่ IP เครื่อง A:
   ```bat
   set "URL=http://192.168.1.50:8888/display/"
   ```
4. ดับเบิลคลิกเปิด → จอแสดงเต็มจอ + เสียงเรียกคิวเล่นได้ทันที (ไม่ต้องแตะจอ)
5. ตั้งเปิดอัตโนมัติ: สร้าง shortcut ไปวางใน `shell:startup`

> มีหลายจอ? เปิดจอที่ 2 ด้วย `start-display.cmd 2` (หรือไฟล์ [start-display-2.cmd](start-display-2.cmd))

## C. ตั้งค่าเครื่องประจำห้อง (เรียกคิว) — เครื่อง C, D, ...

เครื่องของเจ้าหน้าที่ประจำแต่ละห้อง ใช้เรียกคิวลูกค้า

1. เปิดเบราว์เซอร์ (Chrome/Edge) ไปที่ IP เครื่อง A:
   ```
   http://192.168.1.50:8888/calling
   ```
2. **ล็อกอิน** ด้วยผู้ใช้ที่กำหนด (ค่าตั้งต้น: `admin` / `123456` — ควรเปลี่ยน/สร้างผู้ใช้แยกในข้อ D)
3. เลือก **"ห้องที่ประจำการ"** ให้ตรงกับห้องของเครื่องนั้น
4. ใช้ปุ่ม: **เรียกคิวถัดไป ▶ / เรียกซ้ำ 🔁 / พักคิว ⏸ / เสร็จสิ้น ✓ / ข้าม ⏭ / ยกเลิก ✕**

> **แนะนำ:** สร้าง shortcut ของเบราว์เซอร์ที่เปิด URL `…/calling` ตรง ๆ วางที่ Desktop ของแต่ละเครื่องห้อง เพื่อความสะดวก
> เครื่องห้องไม่ต้องติดตั้งอะไรเพิ่ม ขอแค่อยู่ LAN เดียวกันและเปิดเว็บได้

## D. ตั้งค่าระบบครั้งแรก (ผู้ดูแล)

เปิดจากเครื่องไหนก็ได้ผ่านเว็บ (ใช้ IP เครื่อง A):

| หน้า | URL | ใช้ทำอะไร |
|---|---|---|
| ตั้งค่า | `http://192.168.1.50:8888/settings` | เปลี่ยนรหัส `admin` / เพิ่มผู้ใช้, จัดการห้อง/ช่วงเวลา/โควตา, รูปแบบบัตรคิว, อัปโหลดเสียงเรียก & โลโก้ |
| แดชบอร์ด | `http://192.168.1.50:8888/` | ดูภาพรวมคิวทุกห้องแบบเรียลไทม์ |
| รายงาน | `http://192.168.1.50:8888/reports` | ดู/ดาวน์โหลดรายงานคิว |

> **ควรทำก่อนเปิดใช้จริง:** เปลี่ยนรหัสผ่าน `admin` จากค่าตั้งต้น และตั้งชื่อ/สีห้องให้ตรงกับหน้างาน

## E. แก้ปัญหาเบื้องต้น

| อาการ | สาเหตุ / วิธีแก้ |
|---|---|
| เครื่องจอ/ห้องต่อ server ไม่ติด (หน้าว่าง) | 1) server ยังไม่เปิด → เปิด `start-server.cmd` ที่เครื่อง A  2) IP ผิด → ตรวจ `ipconfig` เครื่อง A  3) Firewall บล็อก → ทำข้อ A4 |
| เครื่อง A เองเปิด `localhost:8888` ไม่ได้ | server ไม่ได้รัน หรือ Node ไม่ได้ติดตั้ง — ดูข้อความในหน้าต่าง `start-server.cmd` |
| เปิด `start-server.cmd` แล้วเด้งปิด/ฟ้องไม่พบ Node | ยังไม่ได้ติดตั้ง Node 24 (ข้อ A1) |
| จอภาพขึ้นแต่ไม่มีเสียง | ต้องเปิดผ่าน `start-display.cmd` เท่านั้น (ไม่ใช่เปิดแท็บ Chrome ธรรมดา) — ดูคู่มือจอ |
| Kiosk พิมพ์ไม่ออก | ตรวจชื่อเครื่องพิมพ์ใน `interface` ให้ตรงกับชื่อใน Windows / เครื่องพิมพ์เปิดอยู่ / ลองกด `Ctrl+Shift+P` ทดสอบพิมพ์ |
| IP เครื่อง A เปลี่ยนเอง ทำเครื่องอื่นหลุด | ตั้ง Static IP หรือจอง DHCP Reservation (ข้อ A3) |

## ✅ Checklist ก่อนเปิดใช้จริง

- [ ] เครื่อง A: ติดตั้ง Node 24, เปิด Firewall 8888, ตั้ง Static IP
- [ ] เครื่อง A: `start-server.cmd` เปิดได้ + อยู่ใน Startup + เปิด `localhost:8888` เห็นแดชบอร์ด
- [ ] เครื่องอื่นในวง เปิด `http://<ip เครื่อง A>:8888` ได้ (พิสูจน์ว่า LAN/Firewall ผ่าน)
- [ ] เครื่อง A: Kiosk พิมพ์บัตรคิวออกจริง + อยู่ใน Startup
- [ ] เครื่อง B: จอแสดงผลเต็มจอ + มีเสียงเมื่อกดเรียกคิว + อยู่ใน Startup
- [ ] เครื่องห้องทุกห้อง: เปิด `/calling` เลือกห้องถูก กดเรียกแล้วเลขขึ้นที่จอแสดงผล
- [ ] เปลี่ยนรหัสผ่าน `admin` + ตั้งค่าห้อง/เสียง/บัตรเรียบร้อย
